1. App Not Loading (employees can't open the app)

Check 1 — Is Docker Desktop running?

Look in the taskbar (bottom-right, near the clock) for the Docker whale icon.

  • If you don't see it: Open Docker Desktop from the Start menu. Wait 1–2 minutes for it to fully start.
  • If it shows a red/yellow dot: Docker is starting up. Wait another minute.

Check 2 — Start the app containers

Open a folder window and navigate to the project folder (e.g. D:\Kaivalya or C:\Kaivalya). Right-click in the folder and open PowerShell or Terminal, then run:

docker compose up -d

Wait 30 seconds, then try opening the app again.

Check 3 — Restart the PC

If Docker is running but the app still doesn't load, restart the PC. Docker and all services start automatically on login (within 1–2 minutes).

Check 4 — Internet access is down but local access works

Employees in the office can always access the app at http://localhost:3000. Internet access (via app.vrivalsarena.com) goes through the Cloudflare tunnel. If the tunnel is down, local access still works.

To restart the tunnel, open Command Prompt as Administrator and run:

sc start cloudflared
Still not working after all of the above?
Call your developer (contact details at the end of this document).

2. Something Looks Wrong in the Data

Important: Do NOT delete anything yourself
If quotations, customers, or files appear missing or incorrect, do not attempt to fix the database manually. Call your developer first.

What you can safely do

  • Take a screenshot of what looks wrong.
  • Note the time it was noticed and what was last done before that.
  • Run the health check script to confirm the system is up.
  • Call your developer with these details.

If you need to restore from a backup

This is done by your developer, or you can follow these steps with their guidance on the phone:

1
Navigate to the project folder. Right-click → Open in PowerShell.
2
Run the restore script:
powershell -ExecutionPolicy Bypass -File "backup\restore-appwrite.ps1"
3
A list of available backups will appear, numbered 0, 1, 2. Press Enter to use the most recent backup (recommended), or type a number to choose an older one.
4
Type YES (all capitals) and press Enter to confirm.
5
The script will stop the app, restore all data, and start everything again. This takes about 1–2 minutes. The app will be unavailable during this time.
Restore warning
Restoring from backup replaces ALL current data with the backup copy. Any work done after the backup was taken will be lost. The backup runs daily at 1:00 PM.

3. Low Disk Space Warning

A daily check runs at 9:00 AM. If the C: drive is running low, a notification will appear on the desktop.

NotificationWhat it meansWhat to do
Disk Space WarningBelow 100 GB freeDelete old files, empty Recycle Bin, or contact developer
CRITICAL: Disk Almost FullBelow 50 GB freeAct immediately — the app may stop working

Quick ways to free space

  • Empty the Recycle Bin.
  • Delete files in Downloads folder you no longer need.
  • Run Windows Disk Cleanup (search "Disk Cleanup" in Start menu).
  • Call your developer if unsure — do not delete Docker files or the project folder.

4. Daily Backup

The backup runs automatically every day at 1:00 PM (during lunch break). It saves a copy of all quotations, customers, purchase orders, uploaded files, and settings.

What is backed upWhere
All database data (quotations, customers, users, POs)Project folder → backup\snapshots\
All uploaded files (drawings, PDFs, CAD files)Same location
App configuration and certificatesSame location

The last 3 daily backups are kept. If the PC is off at 1:00 PM, the backup runs automatically when the PC is next turned on.

Running a manual backup

If you want to take a backup right now (e.g. before a restore, or before a major data entry session):

powershell -ExecutionPolicy Bypass -File "backup\backup-appwrite.ps1"

5. PC Must Be On for the App to Work

This is a self-hosted system
The app runs on this PC. If the PC is turned off, nobody can access it — not in the office, and not remotely. Make sure the PC is on during working hours.

The PC does not need to be logged in for the app to work — just turned on. Docker and the app start automatically in the background when Windows starts.

Before shutting down or restarting

  • Let employees know the app will be unavailable for a few minutes.
  • After restarting, wait 2 minutes before telling employees it is ready.
