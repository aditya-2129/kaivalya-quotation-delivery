Manual backup

powershell -ExecutionPolicy Bypass -File "backup\backup-appwrite.ps1"

Restore procedure

powershell -ExecutionPolicy Bypass -File "backup\restore-appwrite.ps1"

Prompts for backup selection and YES confirmation. Stops Appwrite workers, drops and recreates DB, restores volumes, brings everything back up. Takes ~1–2 min.

Manual restore (without the script)

# 1. Stop Appwrite workers (keep mariadb running) docker compose stop appwrite appwrite-worker-audits appwrite-worker-webhooks \ appwrite-worker-deletes appwrite-worker-databases appwrite-worker-builds \ appwrite-worker-functions appwrite-worker-mails appwrite-realtime # 2. Drop and recreate DB docker exec -i appwrite-mariadb mysql -uroot -p"<ROOT_PASS>" -e \ "DROP DATABASE IF EXISTS appwrite; CREATE DATABASE appwrite CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;" # 3. Restore SQL Get-Content "backup\snapshots\<TIMESTAMP>\appwrite-db.sql" | \ docker exec -i appwrite-mariadb mysql -uroot -p"<ROOT_PASS>" appwrite # 4. Restore a volume (repeat for each) docker run --rm \ -v "kaivalya_appwrite-uploads:/data" \ -v "C:\path\to\backup\snapshots\<TIMESTAMP>:/backup:ro" \ alpine sh -c "rm -rf /data/* /data/.[!.]* 2>/dev/null; tar -xf /backup/uploads.tar -C /data" # 5. Start everything docker compose up -d

Setting Up on a Fresh Machine

  1. Install Docker Desktop. In Settings → Resources → Advanced, move disk image to data drive if C: is small.
  2. Install Git.
  3. Clone repo to data drive: git clone <repo-url> D:\Kaivalya
  4. Copy appwrite\.env and frontend\.env from backup (these are gitignored).
  5. From project root: docker compose up -d
  6. Wait ~5 min for images to pull and containers to start.
  7. Restore latest backup: powershell -ExecutionPolicy Bypass -File "backup\restore-appwrite.ps1"
  8. Register scheduled tasks (run as Administrator):
    powershell -ExecutionPolicy Bypass -File "backup\register-backup-task.ps1" powershell -ExecutionPolicy Bypass -File "scripts\register-disk-alert-task.ps1"
  9. Set up Cloudflare Tunnel (see SETUP.md → Production Setup section).
  10. Update start-kaivalya.bat in Windows startup folder with new drive letter.
The two .env files are NOT in git
appwrite\.env and frontend\.env must be copied manually from a secure backup. Without appwrite\.env, the backup/restore scripts will fail. Without frontend\.env, the frontend container will fail to start.

Cloudflare Tunnel

ActionCommand (run as Administrator)
Check statussc query cloudflared
Start tunnelsc start cloudflared
Stop tunnelsc stop cloudflared
Check config pathsc qc cloudflared (look for --config flag in BINARY_PATH_NAME)

Tunnel config: C:\Users\<username>\.cloudflared\config.yml

Service config copy: C:\Windows\System32\config\systemprofile\.cloudflared\config.yml

Common tunnel issue: Error 1033
Service is running but ignores config. Fix: check sc qc cloudflared — if BINARY_PATH_NAME has no --config flag, re-run:
sc config cloudflared binPath= "\"C:\Program Files (x86)\cloudflared\cloudflared.exe\" tunnel --config \"C:\Users\<username>\.cloudflared\config.yml\" run"

Troubleshooting Quick Reference

SymptomDiagnosisFix
All containers down after rebootDocker didn't auto-startCheck start-kaivalya.bat in startup folder; Docker Desktop must be set to start on login
DB restore fails mid-waymariadb container not running, or wrong passwordCheck _APP_DB_ROOT_PASS in appwrite\.env; ensure mariadb container is up
Volume restore produces empty dataWrong volume name (project prefix mismatch)Run docker volume ls to confirm actual volume names; update script if folder was renamed
Frontend can't reach AppwriteNEXT_PUBLIC_APPWRITE_ENDPOINT wrongCheck frontend\.env; rebuild: docker compose up -d --build quotation-maker
Appwrite shows router protection error_APP_OPTIONS_ROUTER_PROTECTION enabledSet to disabled in appwrite\.env; restart: docker compose restart appwrite
API key stopped working silentlyKey expired or was deletedCheck expiry in Appwrite console → Settings → API Keys; regenerate and update frontend\.env
File upload rejected over 30 MBOld storage limitConfirm _APP_STORAGE_LIMIT=100000000 in appwrite\.env; restart appwrite container
Backup log shows ERRORmysqldump failed or volume tar failedCheck Docker is running and mariadb container is up; run backup manually to see live output
Task Scheduler not running backupTask not registered, or PC user changedRe-run register-backup-task.ps1 as Administrator; verify task in Task Scheduler
Port 80 conflictIIS or another web server on port 80Stop IIS: net stop w3svc; or disable in Windows Features
